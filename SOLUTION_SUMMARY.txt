PROBLEMS SOLVED:
1. ✅ SIGSYS "bad system call" crash on Termux/Android
2. ✅ TLS certificate issues on minimal distros (Arch, Alpine, Termux)

ROOT CAUSES:
1. exec.Command("curl") triggered faccessat2 syscall (not available on Android)
2. Native Go HTTP needs TLS certificates from various distro-specific locations

SOLUTION APPROACH:
- Replaced curl with native Go net/http package (eliminates syscall issue)
- Added three-tier TLS certificate loading system (solves cert issue)
- Tier 1: System cert pool (90% of systems)
- Tier 2: Manual loading from common paths (minimal distros)
- Tier 3: Insecure fallback with warning (Termux last resort only)
